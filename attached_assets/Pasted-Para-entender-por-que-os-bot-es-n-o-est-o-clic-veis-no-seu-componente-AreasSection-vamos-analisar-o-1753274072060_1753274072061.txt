Para entender por que os botões não estão clicáveis no seu componente AreasSection, vamos analisar o código e identificar possíveis problemas. O componente usa Link do wouter para navegação e button dentro dele, mas os botões podem não estar funcionando como esperado. Aqui estão as possíveis causas e soluções:

1. Conflito entre Link e button
O componente Link do wouter já é um elemento de navegação que gera uma tag <a> internamente. Colocar um button dentro de um Link pode causar comportamento inesperado, como o botão não responder aos cliques corretamente, devido a conflitos de eventos ou renderização. O Link já lida com a navegação, então o button pode ser redundante.

Solução: Remova o button e estilize o próprio Link para se parecer com um botão. O Link pode receber classes CSS e eventos como onClick.

  onClick={() => setTimeout(() => window.scrollTo(0, 0), 100)}

2. CSS ou Propriedades Bloqueando Interatividade
O CSS pode estar interferindo na interatividade dos botões. Propriedades como pointer-events: none, z-index negativo, ou elementos sobrepostos (como o div com a classe absolute inset-0) podem bloquear cliques.

Verifique o div com gradiente de fundo:

3. Problemas com o wouter ou Configuração de Rotas
Se o Link não está navegando, pode haver um problema com a configuração do wouter. Certifique-se de que:

O componente está dentro de um Router fornecido pelo wouter.
As rotas /agricultura, /agricultura/map, /pesca, /pesca/map, /paa, /paa/map estão definidas corretamente no seu aplicativo.

4. Evento onClick no button
O onClick no button está configurado para executar setTimeout(() => window.scrollTo(0, 0), 100). Isso deve funcionar, mas se o evento de clique não está sendo disparado, pode ser um sinal de que o botão está sendo bloqueado (veja o ponto 2) ou que há um erro no JavaScript.

Solução: Para testar, adicione um console.log no onClick para verificar se o evento está sendo disparado:

jsx

<button
  className="flex items-center text-green-600 font-semibold hover:text-green-700 transition-colors cursor-pointer w-full text-left p-2 rounded-lg hover:bg-green-50"

    console.log("Botão clicado!");
    setTimeout(() => window.scrollTo(0, 0), 100);
  }}
>
Se o console.log não aparecer, o problema está no bloqueio de cliques (provavelmente CSS). Se aparecer, mas a navegação não ocorre, o problema está no wouter.
