Prompt para Ajuste do Sistema de Coordenadas Manuais

Contexto:
No painel administrativo da agricultura, há um componente que permite inserir coordenadas manualmente em três formatos:

Decimal (ex: -2.8792, -52.0088)

Graus e minutos (ex: 2° 52.752'S, 52° 0.528'O)

Graus, minutos e segundos (ex: 2° 48' 6.25"S, 52° 4' 13.74"O)

Problema Identificado:
O sistema está convertendo automaticamente todas as coordenadas para o formato decimal, mesmo quando o usuário insere no formato DMS (graus, minutos, segundos), resultando em:

Conversão incorreta dos valores

Localizações sendo plotadas em lugares errados no mapa

Perda da formatação original que o usuário inseriu

Requisitos para a Solução:

Manter o formato original inserido pelo usuário

Garantir que as coordenadas sejam interpretadas corretamente sem conversão automática

Validar precisão das coordenadas nos três formatos

Plotar no mapa exatamente no local especificado

Armazenar no banco de dados no formato original inserido

Tarefas Específicas:

Modificar a função parseDMS para:

Retornar o valor no formato original quando válido

Adicionar validação rigorosa para cada formato

Manter os sinais de direção (N/S, E/O)

Ajustar o componente IconSelector para:

Exibir o valor formatado corretamente após inserção

Não converter para decimal automaticamente

Manter a formatação original nos campos de input

Atualizar a função de plotagem no mapa para:

Interpretar corretamente cada formato de coordenada

Manter a precisão original

Não realizar conversões desnecessárias

Adaptar o armazenamento para:

Salvar o formato original

Manter metadados sobre o formato utilizado

Garantir compatibilidade com buscas futuras

Exemplo de Comportamento Esperado:

Usuário insere: "2° 48' 6.25"S, 52° 4' 13.74"O"

Sistema:

Valida como formato DMS válido

Mantém exatamente esta formatação

Plota no local exato correspondente

Armazena como string no formato original

Quando recuperado, mostra exatamente: "2° 48' 6.25"S, 52° 4' 13.74"O"

Validações Necessárias:

Para formato DMS:

Graus entre 0-90 (latitude) ou 0-180 (longitude)

Minutos entre 0-59

Segundos entre 0-59.999...

Direção cardinal válida (N/S/E/O)

Para formato DM:

Graus como acima

Minutos entre 0-59.999...

Direção cardinal válida

Para decimal:

Latitude entre -90 e 90

Longitude entre -180 e 180

Notas Adicionais:

Priorizar precisão sobre conversão automática

Manter consistência com sistemas GIS comuns

Documentar os formatos aceitos no componente

Adicionar tooltips com exemplos válidos

Criar testes unitários para cada formato

Resultado Esperado:
Um sistema de coordenadas que respeite o formato inserido pelo usuário, mantenha a precisão geográfica e forneça feedback claro sobre o formato utilizado, sem conversões não solicitadas.
