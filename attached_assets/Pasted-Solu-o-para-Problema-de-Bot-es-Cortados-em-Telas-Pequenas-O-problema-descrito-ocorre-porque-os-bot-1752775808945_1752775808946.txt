Solução para Problema de Botões Cortados em Telas Pequenas

O problema descrito ocorre porque os botões de sugestão estão sendo posicionados com uma altura fixa que não se adapta bem a telas menores (14 polegadas com zoom 100%). Quando o zoom é reduzido para 80%, os botões aparecem porque há mais espaço vertical disponível.
Soluções Propostas:

    Remover altura fixa e usar flexbox dinâmico:
    tsx

{suggestions.length > 0 && (
  <div
    className="p-2 border-t flex flex-wrap gap-2 bg-gray-50 z-10 w-full rounded-b-lg shadow-md"
    style={{
      position: "absolute",
      bottom: "0",
      left: "0",
      right: "0",
      width: "100%",
      zIndex: 10,
      maxHeight: "calc(100% - 70px)", // Ajuste dinâmico
      overflowY: "auto",
    }}
  >
    <div className="w-full flex flex-wrap gap-2">
      {suggestions.map((suggestion, index) => (
        <Button
          key={index}
          variant="outline"
          size="sm"
          className="text-xs bg-white hover:bg-green-50 border-green-200 text-green-800"
          onClick={() => {
            handleSuggestionClick(suggestion);
            setTimeout(() => scrollToBottom(), 100);
          }}
        >
          {suggestion.text}
        </Button>
      ))}
    </div>
  </div>
)}

Ajustar o container principal para ser mais responsivo:
tsx

<Card
  className="w-80 sm:w-96 md:w-[420px] lg:w-[450px] xl:w-96 shadow-xl flex flex-col"
  style={{
    height: isMobile ? "calc(100vh - 32px)" : "580px",
    maxHeight: isMobile ? "none" : "min(580px, 80vh)",
    minHeight: isMobile ? "auto" : "500px",
    position: "relative",
  }}
  data-chat-open="true"
>

Melhorar o posicionamento do formulário de entrada:
tsx

    <form
      onSubmit={handleSubmit}
      className="p-3 border-t flex items-center bg-white z-20 w-full min-h-[70px]"
      style={{
        position: "absolute",
        bottom: suggestions.length > 0 ? (isMobile ? "auto" : "70px") : "0",
        left: "0",
        right: "0",
        width: "100%",
        zIndex: 20,
        borderTop: "1px solid #e5e7eb",
      }}
    >

Mudanças Recomendadas:

    Adicione um estado para detectar telas pequenas:
    tsx

const [isSmallScreen, setIsSmallScreen] = useState(false);

useEffect(() => {
  const checkScreenSize = () => {
    setIsSmallScreen(window.innerWidth < 1024); // ou outro breakpoint adequado
  };
  
  checkScreenSize();
  window.addEventListener('resize', checkScreenSize);
  return () => window.removeEventListener('resize', checkScreenSize);
}, []);

Modifique o estilo dos botões de sugestão para serem mais compactos em telas pequenas:
tsx

<Button
  key={index}
  variant="outline"
  size={isSmallScreen ? "xs" : "sm"}
  className={`text-xs bg-white hover:bg-green-50 border-green-200 text-green-800 ${
    isSmallScreen ? "px-2 py-1" : "px-3 py-2"
  }`}
>
  {suggestion.text}
</Button>

Ajuste o container de mensagens para ter mais espaço quando necessário:
tsx

    <div
      className="flex-1 overflow-y-auto p-4 scrollbar-thin scrollbar-thumb-gray-400 scrollbar-track-transparent"
      style={{
        paddingBottom: suggestions.length > 0 ? "140px" : "70px",
        minHeight: "280px",
        maxHeight: isSmallScreen 
          ? "calc(100vh - 200px)" 
          : "calc(500px - 140px)",
      }}
    >

Essas mudanças tornarão o componente mais responsivo e evitarão que os botões fiquem cortados em telas menores, mantendo uma boa experiência do usuário em todos os tamanhos de tela e níveis de zoom.
