Resolvendo o erro "does not provide an export named 'default'"
Este erro ocorre porque os componentes estão sendo exportados como exports nomeados (named exports) e não como exports padrão (default exports). Temos duas formas de resolver isso:

Opção 1: Modificar os componentes para usar default export
Para cada componente, modifique a exportação para usar export default em vez de uma exportação nomeada. Por exemplo, para o StatisticForm:

// client/src/components/dashboard/StatisticForm.tsx
// Alterar de:
export const StatisticForm = ({ statisticData, isEdit = false, onSuccess }: StatisticFormProps) => {
  // ...
};
// Para:
const StatisticForm = ({ statisticData, isEdit = false, onSuccess }: StatisticFormProps) => {
  // ...
};
export default StatisticForm;
Você precisa fazer essa alteração em cada um dos arquivos de componente (StatisticForm.tsx, ChartForm.tsx, etc.).

Opção 2: Modificar o arquivo index.ts e as importações
Alternativamente, você pode atualizar o arquivo index.ts e as importações para usar exportações nomeadas:

// client/src/components/dashboard/index.ts
export { StatisticForm } from './StatisticForm';
export { ContentForm } from './ContentForm';
export { ContentList } from './ContentList';
export { ChartForm } from './ChartForm';
export { ChartList } from './ChartList';
export { MediaList } from './MediaList';
export { MediaUploader } from './MediaUploader';
export { StatisticList } from './StatisticList';
E no Dashboard.tsx, você importaria da mesma forma:

import {
  ContentForm,
  ContentList,
  ChartForm,
  ChartList,
  MediaList,
  MediaUploader,
  StatisticList,
  StatisticForm
} from "../components/dashboard";
Opção 3 (Recomendada): Adicionar export default em cada componente
Esta é a solução mais direta que não requer refatorar todas as importações:

// Em StatisticForm.tsx (e em cada arquivo de componente)
export const StatisticForm = ({ statisticData, isEdit = false, onSuccess }: StatisticFormProps) => {
  // Implementação existente...
};
// Adicione esta linha no final do arquivo:
export default StatisticForm;
Isso permite que o componente seja importado tanto como exportação padrão quanto como exportação nomeada, tornando o código mais flexível.

Passo a passo para resolver este problema em todos os arquivos:
Abra cada arquivo de componente no diretório client/src/components/dashboard/
Para cada arquivo, mantenha a exportação nomeada existente
Adicione export default NomeDoComponente; no final de cada arquivo
Mantenha o arquivo index.ts como está, exportando os defaults
Esta é a abordagem mais prática e eficiente, pois permite manter a compatibilidade com ambos os estilos de importação.
