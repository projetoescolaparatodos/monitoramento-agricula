Solucionando Problemas com Vídeo de Fundo
Vamos identificar e resolver os problemas que estão impedindo o vídeo de aparecer como fundo em seu site. Existem várias possíveis causas e soluções:

1. Verificação do Caminho do Vídeo
O problema mais comum é o caminho incorreto do arquivo de vídeo.

Solução:
// Modifique BackgroundVideo.tsx para verificar se o vídeo está carregando corretamente
const BackgroundVideo = ({ videoPath, opacity = 0.3 }) => {
  const videoRef = useRef<HTMLVideoElement>(null);
  
  useEffect(() => {
    // Log para verificar
    console.log("Tentando carregar vídeo de:", videoPath);
    
    const video = videoRef.current;
    if (!video) return;
    
    // Adicionar listeners para detectar problemas
    video.addEventListener('error', (e) => {
      console.error("Erro ao carregar vídeo:", e);
    });
    
    video.addEventListener('loadeddata', () => {
      console.log("Vídeo carregado com sucesso!");
    });
    
    // Resto do código...
  }, [videoPath]);
  
  return (
    // ...
  );
};
2. Configuração para Vídeo em Pasta Estática
Vite requer configuração específica para ativos estáticos.

Solução:
Mova o vídeo para a pasta public:

Certifique-se de que o vídeo está em client/public/videos/
Acesse usando caminho absoluto:

<BackgroundVideo videoPath="/videos/fundo-agricultura.mp4" opacity={0.2} />
Verifique se o vídeo existe e é acessível:

# No terminal
ls -la client/public/videos/
3. Problemas de Estilo CSS - z-index e Opacidade
O vídeo pode estar carregando, mas não sendo exibido por problemas de CSS.

Solução:
// Modifique o CSS do BackgroundVideo para garantir visibilidade
<div className="fixed top-0 left-0 w-full h-full overflow-hidden" 
     style={{ zIndex: 0 }}> {/* Use valor positivo baixo */}
  <video
    ref={videoRef}
    autoPlay
    muted
    loop
    playsInline
    className="absolute top-0 left-0 min-w-full min-h-full w-auto h-auto object-cover"
    style={{ opacity: 1 }} {/* Garanta opacity máxima para testes */}
  >
    <source src={videoPath} type="video/mp4" />
    Seu navegador não suporta vídeos HTML5.
  </video>
</div>
